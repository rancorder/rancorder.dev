Web Components完全ガイド

HTML + Web Componentsで作る次世代ブログシステム。MDX不要、依存関係ゼロで実現するインタラクティブコンテンツの全貌

このシステムの特徴

MDXは依存関係が多く壊れやすい。Web Componentsはブラウザネイティブで、依存関係ゼロ、ビルドエラーなし。

このページは、すべてのWeb Componentsを実際に使用したデモです。

2. Code Block（コードブロック）

コピーボタン付きのシンタックスハイライト対応コードブロック：

// Web Components example class CalloutBox extends HTMLElement { connectedCallback() { const type = this.getAttribute('type') || 'info'; this.className = `callout callout--${type}`; } } customElements.define('callout-box', CalloutBox); # Python example def hello_world(): print("Hello, Web Components!") return True if __name__ == "__main__": hello_world() # Bash example npm install npm run build npm start

複数の情報を切り替えて表示：

Web Componentsのメリット

  • 爆速パフォーマンス

Web Componentsのデメリット

  • TypeScript補完が効きにくい
  • SSRに制約がある
  • Reactエコシステムとの統合が限定的

使い方

実装方法

ステップ1: ライブラリを読み込む

<link rel="stylesheet" href="/blog-base.css"> <script type="module" src="/blog-components.js"></script>

ステップ2: コンポーネントを使う

<callout-box type="info" title="タイトル"> 内容を書く </callout-box>

ステップ3: 記事を作成

node scripts/new-post.js "記事タイトル" # → content/blog/YYYY-MM-DD-slug.html が生成される

ステップ4: GitHubにプッシュ

git add . git commit -m "Add new post" git push origin main # → Vercelで自動デプロイ（2-3分）
